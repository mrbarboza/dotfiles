* Doom Settings

Global Doom Emacs settings. Sets the theme to `doom-tokyo-night`, font to `JetBrains Mono` size 15, relative line numbers, disables the kill-emacs confirmation prompt, and maps the `localleader` key to `,.`

#+begin_src emacs-lisp
(setq doom-theme 'doom-tokyo-night)
(setq doom-font (font-spec :family "JetBrains Mono" :size 15))
(setq display-line-numbers-type 'relative)
(setq confirm-kill-emacs nil)
(setq doom-localleader-key ",")

#+end_src

* Markdown

** Headers

Visually customizes Markdown headers using the `variable-pitch` font family, bold weight, and decreasing heights from `1.6` (H1) down to `1.1` (H6), all inheriting from `font-lock-function-name-face`.

(custom-set-faces
 '(markdown-header-face ((t (:inherit font-lock-function-name-face :weight bold :family "variable-pitch"))))
 '(markdown-header-face-1 ((t (:inherit markdown-header-face :height 1.6))))
 '(markdown-header-face-2 ((t (:inherit markdown-header-face :height 1.5))))

 '(markdown-header-face-5 ((t (:inherit markdown-header-face :height 1.2))))
 '(markdown-header-face-6 ((t (:inherit markdown-header-face :height 1.1)))))

** Toggle Markdown View

Defines an interactive function that toggles between `markdown-mode` (editing) and `markdown-view-mode` (read-only preview). Useful for checking the document's rendered appearance without leaving the buffer.

#+begin_src emacs-lisp
(defun mrbarboza/toggle-markdown-view-mode ()
  "Toggle between `markdown-mode' and `markdown-view-mode'."
  (interactive)
  (if (eq major-mode 'markdown-view-mode)
      (markdown-mode)
    (markdown-view-mode)))

#+end_src

** Project Search Directory

Increases the process output buffer to 1MB (improves LSP performance), and — if `~/dev/nu` exists — configures Projectile to search for projects there and disables caching (likely because projects change frequently in Nubank's monorepo).

#+begin_src emacs-lisp
(setq read-process-output-max (* 1024 1024))
(when (file-directory-p mrbarboza/nu-dir)
  (setq projectile-project-search-path (list mrbarboza/nu-dir)
        projectile-enable-caching nil))

#+end_src

** Custom Tools

Configures `lsp-mode` with semantic tokens enabled and a hook that auto-saves the buffer after LSP applies edits (e.g. code actions, renames). Then conditionally loads the internal `nu` package from `nudev` — Nubank's internal dev tooling — if the path exists on disk.

#+begin_src emacs-lisp
(use-package! lsp-mode
  :commands lsp
  :config
  (setq lsp-semantic-tokens-enable t)
  (add-hook 'lsp-after-apply-edits-hook
            (lambda (&rest _)
              (save-buffer))))

(let ((nudev-emacs-path (expand-file-name "nudev/ides/emacs/" mrbarboza/nu-dir)))
  (when (file-directory-p nudev-emacs-path)
    (add-to-list 'load-path nudev-emacs-path)
    (require 'nu nil t)))

#+end_src
